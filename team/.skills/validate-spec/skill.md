# Spec Validation Skill

> **Purpose**: Automatically validate specs generated by PM Agent before passing to Coding Agent
>
> **Effect**: Prevent coding failures due to incorrect specs → Save API calls
>
> **Thariq's Lesson**: "Validation skills can catch 80%+ spec errors before coding"

---

## 🎯 Triggers

### Automatic Trigger
- Auto-pipeline: Immediately after PM Agent completion
- Manual: `bash scripts/run-skill.sh validate-spec PLAN-001`

### Input
- Ticket number (e.g., `PLAN-001`)
- Project path (auto-detected from `.project-config.json`)

### 2. Scope Compliance Check

**Gotcha #1: Prevent Scope Creep**

Compare ticket's Acceptance Criteria with spec content:

1. **Read Ticket**: `planning/tickets/PLAN-{number}-*.md`
2. **Extract Acceptance Criteria**
3. **Extract Feature List from Spec**
4. **Check Out-of-Scope Section**

**Validation Logic**:
```python
# Check if spec contains features not in ticket
spec_features = extract_features_from_spec(spec_file)
ticket_features = extract_acceptance_criteria(ticket_file)

extra_features = spec_features - ticket_features

if extra_features:
    if not has_out_of_scope_section(spec_file):
        ERROR: "Spec contains features not in ticket: {extra_features}"
        ERROR: "Out-of-Scope section is missing."
    else:
        # Check if explicitly mentioned in Out-of-Scope section
        out_of_scope = extract_out_of_scope(spec_file)
        if extra_features not in out_of_scope:
            WARNING: "Additional features not mentioned in Out-of-Scope: {extra_features}"
```

**Keyword-based Check**:
```python
# Scope creep suspect keywords
scope_creep_keywords = [
    "OAuth", "social login", "email verification",
    "password reset", "2FA",
    "admin", "dashboard",
    "statistics", "analytics", "report"
]

# Warn if keyword found in spec but not in ticket
for keyword in scope_creep_keywords:
    if keyword in spec_content and keyword not in ticket_content:
        WARNING: f"Scope creep suspect: '{keyword}' is in spec but not in ticket."
```

### 3. Quality Gates

#### API Spec Quality
- [ ] **REST Convention Compliance**:
  - POST = Create
  - GET = Read
  - PUT/PATCH = Update
  - DELETE = Delete
- [ ] **Endpoint Naming**: `/api/{resource}` format
- [ ] **Error Code Consistency**: INVALID_CREDENTIALS, VALIDATION_ERROR, etc.
- [ ] **Required Error Responses**:
  - [ ] 400 (Bad Request)
  - [ ] 401 (Unauthorized, if auth required)
  - [ ] 404 (Not Found, if applicable)
  - [ ] 500 (Server Error)

#### HTML Wireframe Quality
**Applying Gotcha #4, #5**

- [ ] **No External Libraries**:
  ```bash
  grep -E "tailwind|react|vue|bootstrap|cdn|jsdelivr|unpkg" wireframe.html
  # → ERROR if found
  ```
- [ ] **No Actual API Calls**:
  ```bash
  grep -E "fetch\(|axios\.|XMLHttpRequest|\.get\(|\.post\(" wireframe.html
  # → ERROR if found
  ```
- [ ] **Vanilla JS Verification**:
  ```bash
  grep -E "import.*from|require\(" wireframe.html
  # → ERROR if found
  ```

#### Test Case Quality
**Web-Fullstack/Desktop App**:
- [ ] **Accessibility Tests Included** (Gotcha #9):
  ```bash
  grep -i "accessibility\|keyboard\|screen reader" test-cases/PLAN-*-frontend.md
  # → WARNING if not found
  ```

**CLI Tool**:
- [ ] **Exit Code Definition**:
  ```bash
  grep -i "exit code" specs/PLAN-*-command-spec.md
  # → ERROR if not found
  ```

### 4. Implementation Details Intrusion Check

**Gotcha #10: Prevent Coding Agent Role Intrusion**

Check if spec contains implementation details:

**Forbidden Keywords**:
```python
implementation_keywords = [
    # Libraries/Packages
    "bcrypt", "jwt", "jose", "passport",
    "axios", "fetch", "request",
    "mongoose", "sequelize", "typeorm",

    # File/Directory Structure
    "src/", "lib/", "utils/", "helpers/",
    "controllers/", "services/", "models/",

    # Database Details
    "VARCHAR", "INTEGER", "TIMESTAMP",
    "PRIMARY KEY", "FOREIGN KEY", "INDEX",

    # Design Patterns
    "Singleton", "Factory", "Observer",
    "Repository Pattern", "MVC",
]

for keyword in implementation_keywords:
    if keyword in spec_content:
        WARNING: f"Implementation detail suspect: '{keyword}'"
        WARNING: f"PM Agent defines 'what', Coding Agent decides 'how'"
```

### 5. Auto-fix

Automatically fix minor issues:

#### Fixable Issues

**1. Hardcoded Passwords → Environment Variable Reference**
```python
# Before
"password": "admin123"

# After
"password": "${ADMIN_PASSWORD}"
```

**2. HTTP → HTTPS**
```python
# Before
"url": "http://api.example.com"

# After
"url": "https://api.example.com"
```

**3. Add Missing Out-of-Scope Section**
```markdown
## Out of Scope
(Auto-generated: Manual review required after spec review)
```

## 🔧 Usage

### Auto-pipeline Integration

```python
# After PM Agent execution in auto_pipeline.py
result_pm = self.run_agent("pm", ticket_content, ticket_num)

# Run validation skill
validation = self.run_skill("validate-spec", ticket_num)

if not validation["passed"]:
    print(f"\n⚠️  Spec validation failed:")
    print(f"  Errors: {validation['errors']}")
    print(f"  Warnings: {validation['warnings']}")

    # Apply auto-fixes
    if validation.get("auto_fixes"):
        print(f"\n✓ Applied auto-fixes: {len(validation['auto_fixes'])} items")

    # Re-run PM Agent (with issues)
    retry_prompt = f"""
The previous spec had the following issues:

{format_issues(validation['errors'], validation['warnings'])}

Please regenerate the spec with the above issues fixed.
"""
    result_pm_retry = self.run_agent("pm", retry_prompt, ticket_num)

    # Re-validate
    validation_retry = self.run_skill("validate-spec", ticket_num)

    if not validation_retry["passed"]:
        print("❌ Re-validation failed - Manual intervention required")
        return False

print("✅ Spec validation passed - Proceeding to Coding Agent")
```

### Manual Execution

```bash
# Validate specific ticket
bash scripts/run-skill.sh validate-spec PLAN-001

# Validate all tickets
bash scripts/run-skill.sh validate-spec --all

# Validate + auto-fix
bash scripts/run-skill.sh validate-spec PLAN-001 --auto-fix
```


```
         ↓ (Incorrect spec)
```
